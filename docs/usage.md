# Usage

This repo uses `pnpm` workspaces and TypeScript.

## Use the Core Package

Core is a runtime-agnostic TypeScript state library. A host can use it directly without the HTTP API:

```ts
import { applyCommand, createGame } from "@mtg-engine/core";

let game = createGame({ players: [{ id: "p1", name: "Jair", library: ["Opt"] }] });

game = applyCommand(game, {
  type: "object.move",
  objectId: game.players[0]!.zones.library.objects[0]!.objectId,
  to: { zone: "hand", playerId: "p1" },
}).state;
```

The Hono API below is one optional host around the same core package. The Vite web app uses that API for server-backed debugging.

## Install

```sh
pnpm install
```

## Run the API

```sh
pnpm dev:api
```

The server listens on `http://localhost:3000` by default. Set `PORT` to use another port.

```sh
PORT=4000 pnpm dev:api
```

## Run the Web App

In another terminal:

```sh
pnpm dev:web
```

The Vite app runs on `http://localhost:5173` by default and calls the API at `http://localhost:3000`. Set `VITE_API_URL` to point at another API host.

## API Endpoints

### Health

```sh
curl http://localhost:3000/health
```

Returns:

```json
{ "ok": true }
```

### Create a New Game

```sh
curl -X POST http://localhost:3000/games \
  -H 'content-type: application/json' \
  -d '{ "name": "Friday Commander" }'
```

Notes:

- The API generates the game ID.
- The game name is display metadata and is set at creation.
- New API games start with no players.
- Games are in memory only and are lost when the API process restarts.

Returns:

```json
{
  "game": { "id": "game_...", "name": "Friday Commander", "createdAt": "...", "updatedAt": "..." },
  "view": { "id": "single", "viewMode": "debug", "players": [] }
}
```

### Get a Game

```sh
curl http://localhost:3000/games/game_...
```

Returns `{ "game": ..., "view": ... }`. Unknown game IDs return `404`.

### Apply a Command

```sh
curl -X POST http://localhost:3000/games/game_.../commands \
  -H 'content-type: application/json' \
  -d '{
    "type": "player.add",
    "player": { "id": "auth-user-id", "name": "Jair" }
  }'
```

Returns:

- `game`: game metadata.
- `view`: the new debug game view.
- `event`: the event generated by the command.

### Read Events

```sh
curl http://localhost:3000/games/game_.../events
```

Returns the in-memory event log for that game.

### Stream Events

```sh
curl -N http://localhost:3000/games/game_.../events/stream
```

Streams Server-Sent Events named `game.event` after commands are applied to that game.

## Error Responses

Invalid request bodies return `400` with `error: "invalid_request"`.
Commands that are valid JSON but invalid for the current game state return `400` with `error: "command_error"`.
Unknown game IDs return `404` with `error: "not_found"`.
