# Usage

This repo uses `pnpm` workspaces and TypeScript.

## Install

```sh
pnpm install
```

## Run the API

```sh
pnpm dev:api
```

The server listens on `http://localhost:3000` by default. Set `PORT` to use another port.

```sh
PORT=4000 pnpm dev:api
```

## API Endpoints

### Health

```sh
curl http://localhost:3000/health
```

Returns:

```json
{ "ok": true }
```

### Get Current Game

```sh
curl http://localhost:3000/game
```

Returns the current debug game view.

### Create a New Game

```sh
curl -X POST http://localhost:3000/game/new \
  -H 'content-type: application/json' \
  -d '{
    "players": [
      { "id": "p1", "name": "Jair", "library": ["Opt"] },
      { "id": "p2", "name": "Skyler" }
    ]
  }'
```

Notes:

- A game must have 1 to 4 players.
- Player IDs are optional. If omitted, the engine generates IDs.
- Player life defaults to `40`.
- Initial cards are represented by name only and get generated `cardId` and `objectId` values.

### Apply a Command

```sh
curl -X POST http://localhost:3000/game/commands \
  -H 'content-type: application/json' \
  -d '{
    "type": "player.adjustLife",
    "playerId": "p1",
    "delta": -3
  }'
```

Returns:

- `view`: the new debug game view.
- `event`: the event generated by the command.

### Read Events

```sh
curl http://localhost:3000/game/events
```

Returns the in-memory event log for the current game.

### Stream Events

```sh
curl -N http://localhost:3000/game/events/stream
```

Streams Server-Sent Events named `game.event` after commands are applied.

## Error Responses

Invalid request bodies return `400` with `error: "invalid_request"`.
Commands that are valid JSON but invalid for the current game state return `400` with `error: "command_error"`.
